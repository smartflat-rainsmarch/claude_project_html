# 터치 키오스크 컨텐츠 표시 프로젝트 계획

## 개요
이 프로젝트는 체육관과 학교의 터치 키오스크를 위한 웹 기반 시스템으로, 바닥 지도, 일정, 건강 데이터, 게임, PDF, 비디오, 갤러리, 설문조사 등의 컨텐츠를 인터랙티브하게 표시합니다. 사용자는 예약, 체크인, 컨텐츠 탐색 등의 상호작용을 지원합니다. 시스템은 세 가지 주요 구성 요소로 구성됩니다:
- **ssapi**: 백엔드 API, 데이터베이스 상호작용, 통합을 위한 PHP 서버 코드 (예: FCM 알림, 결제).
- **game**: 터치 키오스크에 최적화된 프론트엔드 컨텐츠 화면, 인터랙티브 맵과 미디어 갤러리 포함.
- **web**: 컨텐츠, 사용자, 설정 관리를 위한 CMS 인터페이스.

주요 기능: 터치 제스처 지원 (드래그, 줌, 탭), 모듈형 컨텐츠 유형, API 기반 데이터 가져오기. 대상 환경: 피트니스 센터와 교육 기관의 물리적 터치 키오스크.

## 아키텍처
- **백엔드 (ssapi)**: 서버 측 로직, 인증, 데이터 저장 (MySQL), 외부 통합 처리. API에는 사용자 관리, 예약, 건강 추적, 파일 업로드 포함.
- **프론트엔드 (game)**: PDF.js 같은 라이브러리를 사용한 HTML/JS/CSS 클라이언트 측 표시, Swiper로 캐러셀, Cocos2d-JS로 게임. 바닥 가이드 뷰어를 위한 커스텀 브리지 포함.
- **CMS (web)**: Bootstrap, Pug/SCSS, npm으로 구축된 관리자 및 사용자 대시보드, 정적 사이트 생성.
- **데이터베이스**: 사용자, 컨텐츠, 예약 등의 테이블이 있는 MySQL (smartflat_html_20250613.sql 참조).
- **통합**: 푸시 알림용 FCM, SMS/결제 게이트웨이, 위치 기반 기능용 비콘.

고수준 다이어그램 (텍스트 기반):
```
[터치 키오스크] --> [game/ (프론트엔드)] --> [ssapi/ (API)] --> [MySQL DB]
                     |                        |
                     v                        v
                [web/ (CMS)]            [외부 서비스 (FCM, SMS)]
```

## 폴더 구조
- `ssapi/`: PHP 백엔드 (API, 인증, 데이터 처리).
- `game/`: 프론트엔드 컨텐츠 (맵, 갤러리, 게임).
- `web/`: CMS 인터페이스 (관리자/사용자 대시보드).
- `PHPMailer-master/`: 이메일 라이브러리.
- `demo/`: 샘플 게임 (예: Flappy Bird, 학교 가사).
- 루트 파일: `index.html`, `smartflat_html_20250613.sql` (DB 덤프), `type_pdf_gallery.html` 같은 컨텐츠 템플릿.

## 설정 지침
1. **환경**: PHP 8+, MySQL, Node.js, Apache/XAMPP 설치. 웹 서버가 프로젝트 루트를 가리키도록 확인.
2. **데이터베이스**: `smartflat_html_20250613.sql`을 MySQL로 가져오기.
3. **종속성**:
   - PHP: 컴포저 없음; PHPMailer 수동 포함.
   - 웹: `web/`에서 `npm install` 실행 후 `npm run build`.
   - 게임: 데모에서 `build.xml` 참조하여 Closure Compiler로 JS 축소.
4. **구성**: 경로, API 키 (예: FCM)용 `ssapi/cmn_var.php` 업데이트. prod/dev 모드용 `DEV_REAL` 설정.

## 개발 단계
1. **단계 1: 백엔드 강화 (ssapi)** - API 보안, 새 컨텐츠 유형용 엔드포인트 추가, 인증 개선 (예: 세션 대신 JWT).
2. **단계 2: 프론트엔드 최적화 (game)** - 터치 UI 강화, 바닥 가이드 통합, 모든 컨텐츠 유형에 제스처 지원 추가.
3. **단계 3: CMS 정제 (web)** - 컨텐츠 업로드, 사용자 관리 기능, 태블릿용 반응형 디자인 추가.
4. **단계 4: 통합 및 테스트** - 구성 요소 연결, 자동화 테스트 추가, 키오스크 하드웨어에서 검증.
5. **단계 5: 배포 및 유지보수** - 키오스크용 패키징, 성능 모니터링, 종속성 업데이트.

타임라인: 단계당 1주 스프린트로 4-6주.

## 테스트
- **단위 테스트**: API 응답 (예: PHP용 PHPUnit).
- **통합 테스트**: 엔드투엔드 흐름 (예: 키오스크를 통한 예약).
- **수동 테스트**: 장치에서 터치 제스처; 전체 화면 모드; 키오스크 OS 호환성 (예: Windows/Android).
- **도구**: API용 Postman; UI용 브라우저 개발자 도구; 확장으로 터치 시뮬레이션.

## 배포
- **키오스크 설정**: 터치 지원 장치에 배포; 키오스크 모드에서 자동 실행 구성.
- **서버**: HTTPS로 웹 서버 호스팅; 정적 자산용 CDN 사용.
