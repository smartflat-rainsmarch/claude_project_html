# SmartFlat CMS 3.0 화면 메뉴 구조도 + 페이지별 기능 명세서

## 1. 문서 개요

본 문서는 기존 SmartFlat CMS의 프로젝트 구조, 관리자 기능, 콘텐츠 템플릿 구조를 기반으로 SmartFlat CMS 3.0의 **관리자 화면 메뉴 구조**와 **페이지별 기능 명세**를 정리한 문서입니다.

기존 시스템은 관리자 CMS(`web/v1/dist/`), 프로젝트별 키오스크 콘텐츠(`game/school/...`), 콘텐츠 템플릿(`game/webpage/type_*.html`) 중심으로 구성되어 있으며, 프로젝트 관리, 전원 예약, 화면 편집, 원격 제어, 다양한 콘텐츠 타입을 이미 지원하고 있습니다. 이를 기반으로 3.0에서는 운영성, 확장성, 모니터링, 배포 관리 중심으로 화면 구조를 고도화하는 것을 목표로 합니다.

---

## 2. 설계 목표

SmartFlat CMS 3.0 관리자 화면은 아래 목표를 만족해야 합니다.

1. **운영 중심 UI**
   - 단순 편집 화면이 아니라 운영 현황을 한눈에 파악할 수 있어야 함
2. **프로젝트 / 콘텐츠 / 기기 / 배포 분리**
   - 핵심 도메인을 독립 메뉴로 분리
3. **버전관리 / 로그 / 모니터링 강화**
   - 운영 이력과 장애 추적 가능
4. **확장 가능한 IA(Information Architecture)**
   - 학교/도서관/공공기관 등 다양한 템플릿 확장 가능
5. **현실적인 전환 가능성**
   - 기존 PHP + Bootstrap + jQuery 구조에서 점진적으로 적용 가능

## 4. 관리자 기본 레이아웃 구조

### 4.1 레이아웃 구성

```text
┌─────────────────────────────────────────────────────────────┐
│ Header                                                      │
│ [로고] [프로젝트 선택] [전역 검색] [알림] [사용자 메뉴]      │
├───────────────┬─────────────────────────────────────────────┤
│ Sidebar       │ Main Content                                │
│               │                                             │
│ 대시보드      │ 페이지 제목                                  │
│ 프로젝트 관리 │ 브레드크럼                                    │
│ 콘텐츠 관리   │ 필터 / 액션 버튼                              │
│ 기기 관리     │ 본문 카드 / 테이블 / 폼 / 차트                │
│ 배포 관리     │                                             │
│ 모니터링      │                                             │
│ 전원/스케줄   │                                             │
│ 로그/감사     │                                             │
│ 설정          │                                             │
└───────────────┴─────────────────────────────────────────────┘
```

### 4.2 공통 UI 원칙

- 상단 헤더는 고정
- 사이드바는 1차 메뉴 중심
- 상세 페이지는 브레드크럼 제공
- 목록 페이지는 검색/필터/정렬 제공
- 모든 주요 변경 작업은 저장 전 확인 모달 제공
- 성공/실패 결과는 Toast + 로그 기록
- 상태값은 색상 뱃지로 표기
- 위험 작업(삭제, 롤백, 전원 OFF)은 2단계 확인

## 10. 핵심 결론

SmartFlat CMS 3.0의 화면 구조는 기존의 단순 편집 중심 CMS에서 벗어나, 아래 5개 축을 중심으로 재구성하는 것이 가장 적절합니다.

1. **프로젝트 관리**
2. **콘텐츠 관리**
3. **기기 관리**
4. **배포 관리**
5. **모니터링 / 로그**

이 구조를 적용하면 다음 효과를 기대할 수 있습니다.

- 운영 현황 가시성 향상
- 배포 및 롤백 체계화
- 기기 장애 대응 속도 향상
- 콘텐츠 버전 및 자산 관리 명확화
- 장기적으로 SaaS형 플랫폼으로 확장 가능
