# 회의실 예약 시스템 설계서 (초안)

> 작성일: 2026-03-20
> 참고: docs/reservation.md (기획서)

---

## 1. 개요

회사 내 회의실(1~6) 예약 시스템. 두 화면으로 구성:
- **View**: 회의실 문 옆 디스플레이 (현재 상태 + 예약 정보)
- **Upload**: CMS 관리자/직원 예약 등록/수정/삭제

---

## 2. 데이터 저장 방식

### 2.1 저장 위치
- 테이블: `tb_home`
- 컬럼: `hm_other` (TEXT, JSON 저장)
- 프로젝트별로 `hm_idx` 기준 조회

### 2.2 JSON 구조 (hm_other)
```json
{
  "reservations": [
    {
      "id": "rsv_1711000001",
      "room_no": 3,
      "team_name": "개발팀",
      "reserved_by": "홍길동",
      "is_hide_reserved_by": false,
      "attendee_count": 6,
      "is_hide_attendee_count": false,
      "start_datetime": "2026-03-20 13:00",
      "end_datetime": "2026-03-20 14:00",
      "meeting_title": "주간 개발 회의",
      "is_hide_meeting_title": false,
      "meeting_subtitle": "신규 기능 점검",
      "is_hide_meeting_subtitle": true,
      "created_at": "2026-03-20 09:10:00"
    }
  ]
}
```

### 2.3 7일 지난 데이터 자동 삭제
- 예약 조회/저장 시 `end_datetime`이 현재 - 7일 이전인 항목 자동 필터링
- API에서 저장 전 cleanup 수행

## 3. API 설계

### 3.1 기존 ssapi 활용
`ssapi/adm_get.php`에 reservation 타입 추가하여 기존 구조 활용

### 3.2 API 엔드포인트

| 액션 | type 값 | 파라미터 |
|------|---------|----------|
| 목록 조회 | `get_reservations` | hm_idx, room_no, date |
| 현재 상태 | `get_reservation_current` | hm_idx, room_no |
| 등록 | `add_reservation` | hm_idx, reservation JSON |
| 수정 | `update_reservation` | hm_idx, reservation JSON |
| 삭제 | `delete_reservation` | hm_idx, reservation_id |

**또는 v3 API 방식:**
```
GET  /api/v3/homes/{hm_idx}          → hm_other JSON 읽기
PUT  /api/v3/homes/{hm_idx}          → hm_other JSON 저장
```
기존 home.php PUT API의 `hm_other` 필드를 활용하면 별도 API 불필요.

## 4. type_reservation_view.html 설계

### 4.1 용도
- 회의실 문 옆 디스플레이 전용
- 직원이 **멀리서 봐도** 상태를 바로 인식
- 자동 갱신 (30초마다)

### 4.2 화면 레이아웃 (세로 1080x1920 기준)

```
┌──────────────────────────────────┐
│         회의실 3                  │  ← 회의실명 (60px)
│     2026.03.20 (목) 13:25        │  ← 현재 날짜/시간 (30px)
│                                  │
│ ┌──────────────────────────────┐ │
│ │                              │ │
│ │      🔴 현재 회의중           │ │  ← 상태 (80px, 색상 변경)
│ │                              │ │
│ │      개발팀                   │ │  ← 팀명 (50px, 볼드)
│ │    13:00 ~ 14:00             │ │  ← 시간 (40px)
│ │                              │ │
│ │    주간 개발 회의              │ │  ← 제목 (35px)
│ │    신규 기능 점검              │ │  ← 부제목 (28px, 연한색)
│ │                              │ │
│ │    홍길동 · 6명               │ │  ← 예약자/인원 (24px)
│ │                              │ │
│ └──────────────────────────────┘ │
│                                  │
│ ┌──────────────────────────────┐ │
│ │  다음 예약                    │ │
│ │  14:30 ~ 15:30 운영팀        │ │  ← 다음 예약 (28px)
│ └──────────────────────────────┘ │
│                                  │
│ ── 오늘 전체 일정 ──             │
│ 09:00~10:00 기획팀     ░░░░     │  ← 지난 시간: 흐리게
│ 13:00~14:00 개발팀 ★   ████     │  ← 현재: 강조
│ 14:30~15:30 운영팀     ▒▒▒▒     │  ← 예정: 보통
│ 16:00~17:00 디자인팀   ▒▒▒▒     │
└──────────────────────────────────┘
```

### 4.3 상태별 색상

| 상태 | 배경색 | 텍스트 | 아이콘 |
|------|--------|--------|--------|
| 현재 회의중 | `#DC2626` (빨강) | 흰색 | 🔴 |
| 현재 사용 가능 | `#16A34A` (초록) | 흰색 | 🟢 |
| 곧 시작 (15분 내) | `#F59E0B` (노랑) | 검정 | 🟡 |
| 오늘 예약 없음 | `#6B7280` (회색) | 흰색 | ⚪ |

### 4.4 지난 회의 처리
- 종료 시간이 지난 회의: `opacity: 0.3` + 취소선
- 현재 회의: 강조 (큰 카드)
- 예정 회의: 보통 표시

### 4.5 자동 갱신
- 30초마다 현재 시간 업데이트
- 1분마다 데이터 갱신 (API 호출)
- 상태 변경 시 자동 전환 (회의중 → 사용가능)

## 5. type_reservation_upload.html 설계

### 5.1 용도
- CMS 내부 또는 별도 페이지에서 예약 등록/수정/삭제

### 5.2 화면 구성

```
┌──────────────────────────────────┐
│ 회의실 예약 관리                  │
├──────────────────────────────────┤
│ 회의실: [1▼][2][3][4][5][6]      │  ← 버튼형 선택
│ 날짜:   [2026-03-20]             │  ← date picker
├──────────────────────────────────┤
│ ┌──── 예약 입력 ────────────┐   │
│ │ 팀명: [개발팀        ]     │   │
│ │ 예약자: [홍길동] □숨김     │   │
│ │ 인원:   [6명  ] □숨김     │   │
│ │ 시작: [13:00]              │   │
│ │ 종료: [14:00]              │   │
│ │ 제목: [주간회의   ] □숨김  │   │
│ │ 부제: [점검      ] □숨김   │   │
│ │                            │   │
│ │ [등록] [수정] [삭제] [초기화]│   │
│ └────────────────────────────┘   │
├──────────────────────────────────┤
│ ── 예약 목록 ──                  │
│ 09:00~10:00 │ 기획팀 │ 정기회의  │
│ 13:00~14:00 │ 개발팀 │ 주간회의  │  ← 클릭 → 폼에 로드
│ 14:30~15:30 │ 운영팀 │ 운영회의  │
└──────────────────────────────────┘
```

### 5.3 유효성 검사
1. 필수: 회의실, 팀명, 시작시간, 종료시간
2. 시작 < 종료 검증
3. 같은 회의실 시간 중복 검사
4. 저장 후 목록 자동 갱신

## 6. 데이터 흐름

```
[Upload 화면]
  │ 예약 등록/수정/삭제
  ▼
[v3 API: PUT /homes/{hm_idx}]
  │ hm_other JSON 업데이트
  ▼
[tb_home.hm_other]  ← JSON 저장
  │
  ▼
[View 화면]
  │ 주기적 데이터 조회
  │ GET /homes/{hm_idx} → hm_other 파싱
  ▼
[회의실 문 옆 디스플레이]
```
